كيف تحمين الموقع (النقاط العملية اللي تمنع أغلب الهجمات)

المهمة: معظم الاختراقات للمواقع الثابتة تكون من: XSS (إدخال سكربت)، رفع ملفات خبيثة، سوء إعداد DNS/HTTPS، أو تعرض حساب GitHub نفسه. فالنّصايح التالية تقلّل المخاطر كثير.

  1. HTTPS دائماً — لا تشتغلي موقع بدون HTTPS. GitHub Pages يوفّره تلقائياً؛ لو استعملتي دومين خاص ففعّلي HTTPS (Let's Encrypt مجاني لو تحتاجين شهادة بنفسك). GitHub DocsLet's Encrypt

  2. Content Security Policy (CSP) — ضابطي سياسة تحدد المصادر المسموح تحميل السكربتات والصور منها (default-src 'self' ...). CSP يقلل خطر XSS. لو مستضيفتك ما تسمح بتعديل headers (مثل GitHub Pages)، استخدمي <meta http-equiv="Content-Security-Policy" content="..."> كحل جزئي. MDN Web Docssite

  3. هيدرز أمنيّة مهمة:

    • Strict-Transport-Security (HSTS) — يجبر المتصفح يستخدم HTTPS.

    • X-Frame-Options: DENY — يمنع clickjacking.

    • X-Content-Type-Options: nosniff — يمنع sniffing للمحتوى.

    • Referrer-Policy و Permissions-Policy.
      تقدر تضيفين هيدرز عن طريق Netlify _headers أو في firebase.json لو تستخدمين Firebase. MDN Web DocsNetlify DocsFirebase

    مثال Netlify _headers:

    python
    /*
  Content-Security-Policy: default-src 'self'; script-src 'self'; object-src 'none';
  X-Content-Type-Options: nosniff
  X-Frame-Options: DENY
  Referrer-Policy: no-referrer-when-downgrade
  Strict-Transport-Security: max-age=31536000; includeSubDomains; preload

    مثال firebase.json (headers):

    json
    {
  "hosting": {
    "public": "public",
    "headers": [
      {
        "source": "/**",
        "headers": [
          {"key":"Strict-Transport-Security","value":"max-age=31536000; includeSubDomains; preload"},
          {"key":"X-Content-Type-Options","value":"nosniff"},
          {"key":"X-Frame-Options","value":"DENY"}
        ]
      }
    ]
  }
}

    (Netlify وFirebase يدعمان إعداد هيدرز بسهولة؛ GitHub Pages لا يتيح هالتحكم بنفس السهولة). Netlify DocsFirebase

  4. SRI للسكريبتات من CDNs — إذا تجيبين مكتبات من CDN (مثلاً jQuery من cdnjs)، استعملي Subresource Integrity integrity عشان المتصفح يتحقق من المحتوى ما تنلّب عليه. MDN Web Docs

  5. نظّفي أي مدخلات (sanitize) لو تستعرضين نص من المستخدم — عدم تنظيف المدخلات يؤدي لـ XSS. راجعي إرشادات OWASP لمنع XSS. OWASP Cheat Sheet Series

  6. لا تخزّنين أسرار في الكود — مفاتيح API أو كلمات السر لازم تكون في server (environment variables) أو في إعدادات hosting (Netlify/Firebase environment vars). ملفات الـ front-end لا تخفي شيء.

  7. DDoS / WAF — لو تتوقعين زحمة أو هجمات، فشغلي Cloudflare (DNS + WAF + rate limiting) يحمي الموقع ويخفف هجمات طبقة التطبيق. Cloudflare Docs

  8. حماية الحسابات: فعّلي 2-factor authentication على GitHub، واستخدمي كلمات مرور قوية وخصّصي وصول للمشاركين فقط. هذا يمنع أن يُسرق الريبو ويُنشر بدلاً منك.

  9. النسخ الاحتياطية: خزّني نسخة محلية من كل PDFs وgit clone من الريبو. لو صار حذف أو غصب، ترجعين النسخة بسرعة.

اختيار الاستضافة (بالمختصر)

  • GitHub Pages — الأفضل للمواقع الثابتة البسيطة (مجاني، سهل، HTTPS). GitHub PagesGitHub Docs

  • Netlify / Vercel — أسهل تحكم بالهيدرز، وظائف serverless صغيرة، فورمز، وCI. Netlify Docs

  • Firebase Hosting + Storage — الأفضل لو تريدين لوحة إدارة ورفع ملفات مع auth وقواعد أمان. Firebase+1

  • S3 + CloudFront — خيار احترافي وقابل للتوسعة، استخدمي presigned URLs للتحميل الآمن. AWS Documentation
